# Weir - Админка

> [English](../en/admin-ui.md) - [Начало работы](getting-started.md) - [Безопасность](security.md)

Админка - это Blazor WebAssembly PWA, оформленная как тёмная ops-консоль "Command Center" (вкладки в
верхней панели, плотные моноширинные таблицы, живой дашборд). Она отдаётся хостом по корневому пути и
общается с admin API под `/admin/api`. Все страницы, кроме входа, требуют вошедшего админа. Это
устанавливаемое PWA (см. [Установка](#установка-pwa)).

## Дашборд

Живой обзор, транслируемый по real-time соединению (хаб SignalR), чтобы отражать состояние сервиса без
опроса; если хаб недоступен, идёт откат на периодический HTTP-опрос:

- **Data connections** (сверху): по карточке на каждую целевую БД - индикатор здоровья (зелёный /
  красный), провайдер, латентность probe и, если подключение недоступно, текст ошибки прямо в
  карточке. Сломанная или неверно настроенная БД видна сразу.
- **Service**: компактные плитки - запросы в секунду, всего запросов, ошибки, доля ошибок, запросы
  в полёте, p50 / p95 / p99 латентность, cache hit ratio и uptime процесса.
- **Throughput and latency**: два компактных графика за недавнее окно.
- **Endpoints**: таблица по эндпоинтам (count, доля ошибок, p50 / p95 / p99, cache hit, последний вызов).

Пока трафика data-plane нет, метрики нулевые, а графики плоские - это ожидаемо; карточки подключений
всё равно показывают, доступна ли каждая БД.

## Эндпоинты

Центральная страница - единый datagrid с фильтрацией, объединяющий объекты БД и эндпоинты, которые на
них навешаны. Выберите **connection** (справа сверху); грид покажет процедуры и функции этого
подключения, и для каждой:

- **статус**-пилюлю - `endpoint`, если маршрут создан; `disabled`, если эндпоинт выключен; или
  `no endpoint`, если объект ещё не опубликован;
- маршрут **endpoint** (метод + путь), если он есть;
- колонку **changes**, которая заполняется после **Sync** и показывает, что добавлено / обновлено /
  удалено;
- **действия** - у созданной строки **Test**, **Edit**, **Sync**, **Delete**; у объекта без
  эндпоинта - **Params** (посмотреть параметры) и **Create** (открыть редактор, предзаполненный из
  объекта с уже discovered-параметрами).

Сортировка по колонкам, фильтр по колонкам и общий быстрый фильтр. Если БД подключения не
интроспектируется, грид всё равно показывает его эндпоинты.

**New endpoint** / **Edit** открывают полноценный редактор: маршрут и объект, result mode, таймаут,
флаг enabled, требуемые скоупы и список параметров (source, direction, тип, размер, precision /
scale, колонки TVP). Секция **Caching** включает/выключает кеширование ответа, задаёт TTL и выбирает,
какие входы формируют ключ кеша - отметьте любые параметры эндпоинта, чтобы **варьировать** кеш по ним,
и при необходимости **варьировать по API-ключу**, чтобы вызовы с разными значениями (или разными
ключами) получали отдельные записи кеша; если не выбрано ничего, на эндпоинт кешируется один общий
ответ. Секция **Logging** задаёт политику логирования запросов на эндпоинт (см. [Логи](#логи)).
Сохранение сразу перезагружает каталог на сервере, так что работающий data plane подхватывает новую
конфигурацию кеша и логирования без перезапуска.

Действия панели и редактора:

- **Test** - консоль на основе формы. Вместо ручного JSON она рендерит по одному типизированному полю
  на параметр (числа, даты, чекбоксы; для табличных параметров - редактор строк) и собирает запрос
  сама; переключатель **Raw JSON** показывает сгенерированное тело для продвинутых. Каждое значение
  направляется в источник параметра, поэтому query/header/claim-параметры тоже проверяются, а не
  только body; ответ показывается отформатированным.
- **Discover from database** - заполняет список параметров (включая колонки TVP) из целевого объекта.
- **Sync** (в строке) и **Sync all** (в панели) - согласовать эндпоинты с текущими сигнатурами
  процедур. Слияние сохраняет привязку каждого параметра и обновляет только производную от базы форму,
  добавляет новые и удаляет исчезнувшие; отчёт показывает, что изменилось, а эндпоинт с исчезнувшим
  объектом помечается, а не стирается.
- **Filter** (над списком) - сузить эндпоинты до тех, что требуют выбранный **scope**, и/или до тех,
  которые сможет вызвать выбранный **ключ** (ключ должен иметь все требуемые скоупы и grant на
  процедуру - ровно так, как авторизует вызов data plane). Счётчик показывает, сколько из общего числа
  подходит, и **Export**, и **OpenAPI** учитывают активный фильтр.
- **Export** / **Import** - переносить определения эндпоинтов между окружениями как JSON. Export
  выгружает текущий отфильтрованный набор.
- **OpenAPI** - выгрузить документ OpenAPI 3.0 из метаданных эндпоинтов. При активном фильтре по ключу
  или скоупу выгружается только эта часть поверхности (аудитория указана в заголовке документа), так что
  потребителю можно отдать ровно те эндпоинты, которые доступны его ключу.

## API-ключи

Список ключей, создание ключа (с выбором его скоупов и прав доступа к процедурам) и отзыв.
Plaintext-ключ показывается один раз, в выделенном блоке, сразу после создания - копируйте его тогда,
позже получить нельзя.

В разделе **Procedure access** добавляются grant'ы, ограничивающие, какие процедуры может вызывать
ключ, по уровням: подключение (сервер + база), схема и объект; используйте `*` для "любой". Оставьте
список пустым, чтобы разрешить все процедуры (с учётом скоупов). Полная модель - в разделе
[Безопасность](security.md).

## Скоупы

Определение и удаление именованных прав, привязываемых к ключам и требуемых эндпоинтами.

## Админы

Список аккаунтов админов и создание новых. У каждого аккаунта есть роль: `Admin` (полный доступ) или
`Viewer` (только чтение) - показывается бейджем со статусом enabled / disabled. `Admin` может сбросить
пароль другого аккаунта прямо в строке. Что может каждая роль - см. [Безопасность](security.md).

## Аккаунт

Открывается по ссылке **Account** в футере боковой панели. Показывает ваш логин и роль и позволяет
сменить свой пароль (любая роль может сменить свой; сначала проверяется текущий пароль).

Здесь же управление вашими **access-токенами** - долгоживущими токенами, которые вызывают admin API
от вашего имени, для скриптов и CI/CD. Задайте токену имя и необязательный срок действия; секрет
показывается один раз при создании, скопируйте его сразу. Передавайте его как
`Authorization: Bearer weadm_...`. Отозвать токен можно здесь в любой момент. Как токены соотносятся с
вашей ролью и пример для CI/CD - см. [Безопасность](security.md).

## Аудит

Таблица недавних записей аудита (входы админов, изменения эндпоинтов / ключей / scope / админов /
настроек и прочее), новые сверху. Каждое административное действие пишется с детализацией, что именно
изменилось: `settings.update` перечисляет изменённые поля со старым и новым значением, `endpoint.upsert`
называет метод, маршрут и целевой объект, `endpoint.sync` подытоживает добавленные, обновлённые и
удалённые параметры, и так далее.

## Логи

Экран **Logs** - это история запросов data-plane: по строке на вызов, новые сверху, со временем,
методом, маршрутом, целевым объектом, статусом, полной длительностью, временем БД, числом строк и
вызвавшим ключом. Это инструмент, чтобы понять, "какой запрос был медленным и почему".

- **Пометка медленных** - вызов помечается медленным (предупреждающий маркер у длительности), если он
  превысил скользящее среднее эндпоинта на пороговый процент. Порог берётся из глобальной настройки и
  может переопределяться на эндпоинт. Переключатели **Slow only** и **Errors only** фильтруют такие строки.
- **Переход** - клик по эндпоинту в ленте на **Dashboard** или действие **Logs** в списке
  **Endpoints** открывает этот экран, уже отфильтрованный по эндпоинту.
- **Детали** - клик по строке открывает панель с полным таймингом (включая среднее по эндпоинту для
  контекста) и, для эндпоинтов с включённым захватом, записанные **параметры** запроса и **результат**
  ответа. Захват параметров и результата выключен по умолчанию (может содержать PII) и включается на
  эндпоинт в секции **Logging** редактора эндпоинта (там же задаётся персональный порог медленных).
  Захват результата ограничен по размеру.

## Настройки

Системные настройки рантайма, редактируемые без перезапуска (нужна роль Admin; изменения аудируются):

- **Лимиты data-plane** - лимит строк, таймаут запроса и лимит строк TVP.
- **Rate limiting** - дефолтный лимит на ключ для ключей без своего.
- **Request log** - главный переключатель логирования запросов, глобальный порог медленных (процент
  сверх среднего по эндпоинту) и сколько дней хранить историю запросов.
- **Аудит** - сколько дней хранить историю аудита (фоновый сервис чистит старые записи; ноль - хранить
  вечно).

Экран также показывает значения только для чтения, требующие перезапуска: лимит размера тела запроса и
конфигурацию файлового логирования (директория, ротация, retention, формат, уровень). Они задаются в
конфигурации (`Weir:Logging`, `Weir:DataPlane`) и применяются на старте. См. [Конфигурация](configuration.md).

## Установка (PWA)

Админка - устанавливаемое Progressive Web App. В поддерживающем браузере используйте кнопку установки в
адресной строке (или меню браузера), чтобы добавить **Weir** как отдельное приложение со своей иконкой.
При деплое новой версии появляется всплывающее уведомление (snackbar, снизу справа, в теме приложения)
с кнопкой **Update** - один клик активирует новую сборку и перезагружает на неё; при потере связи
появляется offline-баннер (приложение показывает последние загруженные данные). Чтобы уведомление об
обновлении срабатывало, хост отдаёт управляющие файлы PWA (`service-worker.js`, `service-worker-assets.js`,
`register-sw.js`, манифест) с `Cache-Control: no-cache`, чтобы браузер всегда видел свежий деплой; то же
правило поставьте на reverse-proxy или CDN перед Weir.

## Тема

Интерфейс использует одну фиксированную тему: геометрия Visual Studio 2026, перекрашенная палитрой
**Command Center** (циан на почти чёрном, только тёмная). Сервис тем Flare применяет её до первой
отрисовки, поэтому мигания нет. Верхняя навигация и статус-панель зафиксированы, прокручивается только
содержимое.
